BMv2 is a C++ framework that takes a JSON file compiled from your P4 program and interprets it to implement the packet-processing behavior you defined. It is the standard tool for P4 development — letting you iterate on your data plane without recompiling any C++ code.

<Warning>
  BMv2 is **not** a production-grade software switch. It is designed for developing, testing, and debugging P4 data planes. Throughput and latency are significantly lower than production switches like Open vSwitch.
</Warning>

## How BMv2 works

BMv2 decouples the switch architecture from any specific P4 program. You compile your P4 source once to a JSON representation, then load that JSON into a running bmv2 process — no C++ recompilation needed.

<Steps>
  <Step title="Write your P4 program">
    Write a P4_16 program targeting the `v1model` architecture (or P4_14 for legacy programs).
  </Step>
  <Step title="Compile to JSON">
    Use the `p4c` compiler to produce a BMv2 JSON file:
    ```bash
    p4c --target bmv2 --arch v1model --std p4-16 my_program.p4
    ```
  </Step>
  <Step title="Run the switch">
    Start `simple_switch` with the compiled JSON and bind it to network interfaces:
    ```bash
    sudo simple_switch -i 0@eth0 -i 1@eth1 my_program.json
    ```
  </Step>
  <Step title="Program the control plane">
    Use the CLI or P4Runtime to install table entries, configure multicast groups, and more:
    ```bash
    simple_switch_CLI --thrift-port 9090
    ```
  </Step>
</Steps>
